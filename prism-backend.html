05 — Deployment Infrastructure Options

Deployment Options

Option 01
Docker Compose
All-in-one single-server setup. Good for development and small-scale deployments.
+Simple setup & management
+All services in one place
−Single point of failure
−Limited horizontal scaling
Recommended
Option 02
Kubernetes
EKS, GKE, or self-managed. Stateless Django pods, auto-scaling Celery workers.
+Horizontal auto-scaling
+High availability
+Production-grade resilience
−Higher operational complexity
Option 03
AWS ECS + RDS
ECS Fargate services behind ELB. Managed RDS, SQS, S3 for a fully AWS-native stack.
+Fully managed AWS services
+Auto-scaling built in
−AWS vendor lock-in
−Cost can escalate at scale
06 — Startup Application Bootstrap

Application Startup

01
Run database migrations
python manage.py migrate
02
Collect static files
python manage.py collectstatic --noinput
03
Create superuser (once)
python manage.py createsuperuser
04
Start Django via Gunicorn
gunicorn stellar_backend.wsgi:application --bind 0.0.0.0:8000 --workers 4
05
Start Celery worker(s)
celery -A stellar_backend worker -Q stellar-automation-workflow,stellar-strategy-scheduler --loglevel=info
06
Start Celery Beat — ONE instance only!
celery -A stellar_backend beat --loglevel=info
07 — Checklist Pre-Deployment

Pre-Deployment Checklist

Generate strong SECRET_KEY
Set DEBUG=False in production
Configure ALLOWED_HOSTS — not ['*']
Set up PostgreSQL database with automated backups
Choose and configure message broker (SQS or Redis)
Create .env file with all required variables
Test database migrations: python manage.py migrate
Configure all OAuth credentials (Amazon, Google, Meta, TikTok)
Enable SSL/TLS on database connection
Verify health endpoint responds: GET /health
08 — Scaling Growth Roadmap

Scaling Strategy

Phase 1 · MVP

Single Server

  • Django + Celery on one VM
  • PostgreSQL on separate VM
  • Redis or SQS for queues
  • Minimal ops overhead
Phase 2 · Growth

Load Balanced

  • Load balancer + multi API instances
  • Dedicated Celery worker VMs
  • Managed RDS PostgreSQL
  • Monitoring & alerting
Phase 3 · Enterprise

Kubernetes

  • K8s cluster with auto-scaling
  • Separate node pools per service
  • Database read replicas
  • Distributed caching layer

10 — Security Best Practices

Security Best Practices

01

Secrets Management

Use AWS Secrets Manager or HashiCorp Vault. Never commit .env to git. Rotate API keys regularly. JWT access tokens expire in 5 minutes by default.

02

Database Hardening

Enable SSL/TLS connections. Restrict database access via VPC security groups. Enable automated daily backups with point-in-time recovery.

03

API Security

Enable CORS for trusted domains only. Implement rate limiting. Use short-lived JWT access tokens. Validate all OAuth callback URLs.

04

Infrastructure

Use VPN for admin access. Enable WAF on load balancer. Apply regular security patches. Restrict SSH access via security groups.
